### 3.3 数据库 Schema

```sql
-- 非洲53国白名单
CREATE TABLE africa_countries (
    id          INTEGER PRIMARY KEY,
    code        TEXT UNIQUE NOT NULL,    -- ISO 3166-1 alpha-2
    name_zh    TEXT NOT NULL,
    name_en    TEXT NOT NULL,
    in_afcfta  BOOLEAN DEFAULT TRUE,    -- 是否参与AfCFTA
    has_epa    BOOLEAN DEFAULT FALSE,   -- 是否有欧盟EPA
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- HS编码数据库（非洲热门品类）
CREATE TABLE hs_codes (
    id          INTEGER PRIMARY KEY,
    hs_4        TEXT NOT NULL,          -- 4位HS税号
    hs_6        TEXT,                    -- 6位HS税号
    hs_8        TEXT,                    -- 8位HS税号
    hs_10       TEXT UNIQUE,            -- 10位HS税号（完整）
    name_zh    TEXT NOT NULL,           -- 中文品名
    name_en    TEXT,                    -- 英文品名
    mfn_rate   REAL NOT NULL,          -- 最惠国税率（0-1）
    vat_rate   REAL DEFAULT 0.13,      -- 增值税率
    category   TEXT,                    -- 品类分类
    notes       TEXT,                    -- 备注
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 政策规则表
CREATE TABLE policy_rules (
    id          INTEGER PRIMARY KEY,
    market      TEXT NOT NULL,          -- 'CN'/'EU'/'AFCFTA'
    rule_type   TEXT NOT NULL,          -- 'tariff'/'origin'/'quota'
    hs_pattern  TEXT,                   -- HS编码匹配模式（如 '0901%'）
    rule_text   TEXT NOT NULL,          -- 规则描述
    rate        REAL,                    -- 税率（若是关税规则）
    effective_date DATE,
    source_url  TEXT,                    -- 官方文件链接
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 用户/会员表
CREATE TABLE users (
    id          INTEGER PRIMARY KEY,
    email       TEXT UNIQUE,
    wechat_id   TEXT,                   -- 微信ID
    tier        TEXT DEFAULT 'free',    -- 'free'/'pro'/'enterprise'
    subscribed_at DATETIME,
    expires_at  DATETIME,
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 计算历史（用于数据积累）
CREATE TABLE calculations (
    id          INTEGER PRIMARY KEY,
    user_id     INTEGER,
    product_name TEXT NOT NULL,
    hs_code     TEXT,
    origin      TEXT,                   -- 原产国
    destination TEXT,                   -- 目的地市场
    fob_value   REAL,                  -- FOB货值
    freight     REAL,                  -- 运费
    insurance   REAL,                  -- 保险
    tariff_rate REAL,                  -- 适用税率
    total_cost  REAL,                  -- 总成本
    result_json TEXT,                  -- 完整结果JSON
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 五、关税计算引擎详解

### 5.1 中国零关税计算逻辑

```
输入: HS编码(10位), 原产国(ISO code), FOB货值(USD), 数量(kg)
输出: 关税税率, 关税金额, 增值税, 总进口成本

流程:
1. 验证原产国是否在53国白名单内
   → 不在: 返回"不享受零关税"
2. 查询HS编码对应的最惠国(MFN)税率
3. 判断原产地认定:
   A. 完全原产 → 零关税
   B. 非完全原产:
      - 税号4位改变 → 零关税
      - 增值≥40% → 零关税
4. 计算:
   - 关税 = FOB × 适用税率(0)
   - 增值税 = (FOB + 运费 + 保险 + 关税) × 13%
   - 消费税 = 按品类（如珠宝/酒类）适用
5. 总成本 = FOB + 运费 + 保险 + 关税 + 增值税 + 消费税
```

### 5.2 欧盟EPA估算逻辑

```
输入: 产品HS编码, 原产国, 增值比例
输出: EPA税率估算

规则:
- 必须在EPA参与国（原产国确认）
- 增值≥40% → 零关税
- 增值<40% → 适用MFN税率（需查询欧盟税率表）
- 敏感产品清单: 税率不归零

注意: V1为规则估算，非欧盟官方数据，需加免责声明
```

### 5.3 成本精算器逻辑

```
输入:
- 品种: 埃塞俄比亚耶加雪菲（固定品类）
- 采购量: 20kg（用户输入）
- FOB单价: 6 USD/kg（用户输入）

固定参数（硬编码在数据库中）:
- 国际运费: 约 4-6 USD/kg（埃塞→中国，海运）
- 清关杂费: 约 1500-3000 元/票（固定成本分摊）
- 烘焙损耗率: 约 15%（生豆→熟豆）
- 分装成本: 约 0.5 元/包（227g装）
- 国内物流: 约 5 元/单

输出:
- 到岸总成本 = FOB + 运费 + 清关 + 关税(0) + 增值税
- 每包227g熟豆的原料成本
- 建议零售价（参考市场数据 × 1.5利润系数）
- 回本测算：需要卖多少包
- 原产地证书办理流程（分步骤指南）
```

## 六、API 设计

### 6.1 后端接口

```
POST /api/v1/calculate/tariff
  输入: { hs_code, origin_country, destination, fob_value, currency }
  输出: { tariff_rate, tariff_amount, vat, total_cost, breakdown }

POST /api/v1/calculate/import-cost
  输入: { product_name, quantity_kg, fob_per_kg, origin }
  输出: { total_cost, cost_per_unit, suggested_price, payback_qty, doc_guide }

GET  /api/v1/hs-codes/search?q=咖啡&limit=10
  输入: 查询参数
  输出: [{ hs_10, name_zh, mfn_rate, category }]

POST /api/v1/origin/check
  输入: { product_name, hs_code, origin, processing_steps, material_sources }
  输出: { qualifies, rule_applied, confidence, reasons }

POST /api/v1/subscribe/check
  输入: { email or wechat_id }
  输出: { tier, expires_at, remaining_queries }

GET  /api/v1/countries?market=CN
  输出: [{ code, name_zh, in_afcfta, has_epa }]
```

### 6.2 错误处理

```
 tariff_calculation_error:
   - HS编码不在数据库中 → 提示手动查询+推荐相近编码
   - 原产国不在白名单 → 明确提示"斯威士兰不在零关税范围内"
   - FOB为0或负数 → 输入校验拒绝
   - Claude API超时 → 返回本地规则引擎结果+"AI分析暂不可用"

 origin_check_error:
   - 描述不够详细 → 引导用户提供更多信息
   - Claude API拒绝 → 返回规则判断+"AI无法判定，请咨询专家"
```

## 十、护城河分析

### 10.1 为什么这个产品能建立护城河

**短期（1-6个月）**：先发优势
- 没有人做一个"非洲进口决策工具"
- 内容发布早 = SEO排名先占

**中期（6-12个月）**：内容积累
- 每篇深度指南（选品、准入、合规）= SEO长尾词被占领
- 用户计算历史数据 = 知道什么品类在算、什么放弃了
- 这些数据本身有价值（知道市场需求在哪里）

**长期（12个月+）**：供应商锁定
- 供应商黄页做起来后，用户算完成本直接联系供应商
- 工具变成"查关税+找货源"的双入口
- 迁移成本: 换一个工具 = 失去历史记录 + 重新学

### 10.2 竞品复制难度

| 维度 | 复制难度 | 说明 |
|---|---|---|
| 关税计算逻辑 | 低 | 规则公开，但实现需要研究 |
| 前端界面 | 低 | React代码没有壁垒 |
| 内容积累 | **高** | SEO需要时间，内容壁垒难复制 |
| 供应商数据库 | **高** | 需要线下资源，不能靠爬虫 |
| 用户信任 | **高** | 工具准确 = 用户愿意付费 |
| 计算历史数据 | **高** | 数据越积累越有价值 |
